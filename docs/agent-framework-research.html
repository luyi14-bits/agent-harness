🔬 重点框架深度分析
Superpowers
76k+ stars MIT v6.0.2 2025.10 创建
一句话定位：「An agentic skills framework & software development methodology that works」— 技能驱动的 Agent 软件开发框架，与你的需求几乎完全吻合。
核心工作流：
brainstorming
→
writing-plans
→
git-worktrees
→
subagent-driven-dev
→
TDD
→
code-review
→
finishing-branch
✓ 每个 Skill 自动触发，无需手动调用
✓ 支持子 Agent 驱动开发（subagent-driven-development）
✓ 内置 TDD 红绿重构流程
✓ 支持并行 Agent 调度（dispatching-parallel-agents）
✓ 跨平台支持：Claude Code, Codex, Cursor, Gemini CLI 等 10+
✓ 内置 writing-skills 技能，可创建自定义技能
✓ 系统化调试方法论（systematic-debugging）
✓ 代码审查两阶段评审（spec + quality）
优势
  • 与你的需求高度匹配：skill → agent 工作流
  • 社区极其活跃，9 个月内从 0 到 76k stars
  • 跨平台兼容，不绑定特定 AI 编程工具
  • MIT 协议，可自由 fork 和商用
  • 已有完整的技能生态（brainstorming → plan → code → test → review）
局限
  • 主要面向 AI 编程工具（Claude Code 等），非独立 Agent 运行时
  • 技能以 Shell/JS 为主，非 Python 框架
  • 不自带 LLM 调用层，依赖宿主工具的模型能力
  • 不适合需要独立部署为后端服务的场景
MetaGPT
40k+ stars MIT 2023.06 创建
一句话定位：「The Multi-Agent Framework: First AI Software Company」— 通过 SOP（标准操作程序）指导多 Agent 协作，模拟完整软件公司。
✓ 角色化 Agent：PM、架构师、工程师、QA
✓ SOP 驱动协作，结构化输出
✓ 支持自然语言→可运行代码
✓ Python 框架，可独立部署
优势
  • SOP 概念与你的 tree-sop 理念天然契合
  • Python 实现，可独立运行和部署
  • 角色分工明确，代码生成质量较高
局限
  • 角色固定，灵活性有限
  • 不支持自定义 skill → agent 转换
  • 更新频率近期放缓（最后 push: 2026-01）
CrewAI
25k+ stars MIT 2023.10 创建
一句话定位：角色驱动的多 Agent 协作框架，像组建人类团队一样组建 Agent 团队。
✓ Role + Goal + Backstory 角色定义
✓ 顺序/并行/层级三种调度模式
✓ 原生结构化输出（JSON/Markdown/CSV）
✓ Agent 市场，社区模板可复用
优势
  • 开发效率高，代码量少（比 AutoGen 少 40%）
  • 角色抽象完善，贴合人类团队直觉
  • 社区活跃，更新频繁
局限
  • 人类在回路支持弱（开源版）
  • 不支持自定义 skill → agent 转换
  • 流程可控性一般，不支持复杂分支
LangGraph
22k+ stars MIT 2023.08 创建
一句话定位：基于状态机的 Agent 编排框架，将 Agent、工具、函数都抽象为图的节点。
✓ 图结构编排，支持循环/分支/暂停/恢复
✓ 原生状态持久化和检查点
✓ LangSmith 全链路可观测性
✓ 生产环境最稳定的选择
优势
  • 流程可控性最强，适合生产环境
  • 可自定义任意复杂的工作流逻辑
  • 生态完善，与 LangChain 全栈打通
局限
  • 学习曲线陡峭，概念复杂
  • 需要自己实现 skill → agent 逻辑
  • 代码量较大，样板代码多
OpenHands
72k+ stars MIT 2024.03 创建
一句话定位：开源版 Devin — AI 驱动的自主软件开发 Agent 平台。
✓ 可执行任何人类开发者能做的操作
✓ 代码编写、终端操作、浏览器交互
✓ PR 管理、代码审查
✓ Docker 沙箱安全执行
优势
  • 功能最完整的软件开发 Agent
  • 社区庞大，仅次于 LangChain
  • 可直接用于实际项目开发
局限
  • 单体 Agent，非 skill → agent 架构
  • 不支持自定义 skill 体系
  • 较重，部署成本高

🧭 选型决策树
1
你的核心需求是否是「将 skill 转为独立 agent 进行软件开发」？
→ 是 → 进入步骤 2
→ 否，只需要通用多 Agent 协作 → 选 CrewAI 或 AutoGen
2
你是否需要独立部署为后端服务（而非依赖 Claude Code 等宿主）？
→ 不需要，在 AI 编程工具内使用 → 直接用 Superpowers
→ 需要独立部署 → 进入步骤 3
3
你希望以哪种方式实现 skill → agent？
→ Fork Superpowers + 用 Python 重写核心 → 推荐方案
→ 基于 MetaGPT 的 SOP 架构扩展自定义 skill → 备选方案 A
→ 基于 LangGraph 从零搭建 skill → agent 映射层 → 备选方案 B
→ 基于 CrewAI 将每个 skill 定义为一个角色 Agent → 备选方案 C
4
你的 skill 体系是否结构化（如 tree-SOP 树形结构）？
→ 是，有结构化 skill 定义 → 设计 skill → agent 自动映射机制
→ 否，skill 还在整理中 → 先用 Superpowers 的 writing-skills 方法论梳理
💡 最终建议

推荐方案：Fork Superpowers + 定制化改造

核心理由：Superpowers 是目前开源社区中唯一一个将「skill 体系」与「软件开发方法论」深度融合的框架，其设计理念与你的需求高度一致。它的 brainstorming → planning → subagent-driven-development → TDD → code-review 工作流，本质上就是将一系列 skill 编排为 Agent 协作流程。

具体实施路径：

1. 短期（1-2 周）：直接安装 Superpowers 到你的 AI 编程工具中，体验其 skill → agent 工作流，理解其架构设计

2. 中期（2-4 周）：参考 Superpowers 的 skill 结构（SKILL.md 格式），将你的 tree-sop skill 体系映射为 Superpowers 兼容的 skill 包

3. 长期（1-2 月）：如需独立部署，用 Python 重写核心调度逻辑，参考 MetaGPT 的 SOP 架构和 LangGraph 的状态机编排
