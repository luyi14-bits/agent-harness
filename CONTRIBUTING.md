# Contributing to Jig

Thank you for your interest in contributing to Jig — the first pre-execution Agent Firewall.

## How to Contribute

1. **Fork** the repository
2. **Create a branch** (`git checkout -b feat/your-feature`)
3. **Make your changes**
4. **Run tests** (`pytest tests/ -q`)
5. **Commit** with clear message
6. **Push** and open a Pull Request

## Code Standards

- Python 3.10+ with type annotations
- All public API requires docstrings
- Tests required for new features
- No `except: pass` (security policy)

## Pull Request Process

1. Ensure all tests pass
2. Update CHANGELOG.md for user-facing changes
3. Keep PRs focused on a single concern
4. Reference related issues

## Questions?

Open a [Discussion](https://github.com/luyi14-bits/jig/discussions) or [Issue](https://github.com/luyi14-bits/jig/issues).
